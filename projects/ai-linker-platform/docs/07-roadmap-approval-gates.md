# 07. 단계별 로드맵과 승인 게이트

## 진행 방식

각 단계는 다음 순서로 진행합니다.

```text
계획 확인
 → 화면/기능 제작
 → 로컬 실행 확인
 → 사장님 피드백
 → 수정
 → 승인
 → 다음 단계 진행
```

## Gate 0. 기획/설계 승인

산출물:

- 비즈니스 모델 문서
- 서비스 프로세스 문서
- 모듈 개발계획
- UX 사이트맵
- 관리자 기능계획
- 데이터 모델 초안

승인 기준:

- 사장님이 전체 방향과 기능 우선순위에 동의
- 먼저 만들 화면 범위 확정

현재 상태:

```text
진행 중
```

---

## Gate 1. 프로젝트 생성 / 디자인 시스템

산출물:

- Next.js 프로젝트
- 기본 라우팅
- 공통 레이아웃
- 디자인 토큰
- 버튼/카드/배지/테이블 컴포넌트
- PC/모바일 반응형 구조

승인 기준:

- 웹서비스 첫 화면이 열림
- 관리자 레이아웃이 열림
- 디자인 톤 승인

예상 결과:

```text
projects/ai-linker-platform/web
```

---

## Gate 2. 사용자 웹 MVP

산출물:

- 랜딩 페이지
- Agent 탐색 페이지
- Agent 상세 페이지
- 토큰 충전 소개 페이지
- Skill 마켓 목업
- 커뮤니티 목업

승인 기준:

- 사용자가 쇼핑몰처럼 Agent를 탐색하는 흐름 확인
- Hermes 상품을 예시로 확인 가능

---

## Gate 3. 관리자 대시보드 MVP

산출물:

- 관리자 홈
- 매출/사용자/구매/사용량 카드
- 최근 주문
- 미처리 상담
- LLM 사용량 요약

승인 기준:

- 슈퍼관리자가 서비스 상태를 한 화면에서 파악 가능

---

## Gate 4. 상품/릴리즈 관리

산출물:

- Agent 상품 CRUD UI
- Windows/iOS 릴리즈 관리 UI
- 설치파일 업로드 UI 목업 또는 실제 업로드
- 최신 버전 지정

승인 기준:

- Hermes 설치파일을 상품 릴리즈로 등록 가능

---

## Gate 5. 회원/구매/설치코드

산출물:

- 회원가입/로그인
- 마이페이지
- 구매내역
- 설치코드 발급
- 라이선스 상태

승인 기준:

- 테스트 사용자가 상품 구매 후 설치코드를 확인 가능

---

## Gate 6. 결제/토큰 충전

산출물:

- 결제 연동 또는 결제 목업
- 토큰/크레딧 지갑
- 충전 상품
- 사용량 차감 구조
- 관리자 결제관리

승인 기준:

- 테스트 결제로 크레딧 충전 흐름 확인

---

## Gate 7. LLM Gateway MVP

산출물:

- OpenAI-compatible API endpoint
- AI Linker API Key
- 사용자 인증
- 사용량 기록
- 잔액 차감
- Provider 계정 Pool 관리

승인 기준:

- Agent가 AI Linker Gateway를 통해 LLM 호출 가능
- 사용량과 비용이 관리자 화면에 반영됨

---

## Gate 8. 상담/커뮤니티/채팅

산출물:

- 커뮤니티 게시판
- Q&A
- 상담 티켓
- 관리자 답변
- 채팅 UI

승인 기준:

- 사용자가 문의를 남기고 관리자가 처리 가능

---

## Gate 9. 설치파일 서버 연동

산출물:

- 설치코드 검증 API
- 설치프로그램 서버 인증
- 결제/라이선스 상태 확인
- 최신 릴리즈 확인
- 업데이트 안내

승인 기준:

- Hermes 설치파일이 AI Linker 서버와 연결됨

---

## Gate 10. 운영/보안/배포

산출물:

- 권한 관리
- 감사 로그
- 에러 로깅
- 배포 환경
- 백업 정책
- 개인정보/결제 보안 점검

승인 기준:

- 테스트 배포 환경에서 핵심 플로우 정상 작동

## 바로 다음 단계

사장님 승인 후 Gate 1로 이동합니다.

Gate 1에서 할 일:

1. `projects/ai-linker-platform/web` 생성
2. Next.js + TypeScript + Tailwind 기반 웹앱 구성

4. AI Linker 디자인 시스템 1차 구현
5. 랜딩/관리자 대시보드 기본 화면 제작
