# V-Claw Managed Agent Context

This file is generated by V-Claw at first bootstrap to align OpenClaw runtime paths.
If you customize this file, V-Claw will preserve your changes and will not overwrite it.

## Managed Path Contract

- OpenClaw home env: `OPENCLAW_HOME`
- OpenClaw state env: `OPENCLAW_STATE_DIR`
- OpenClaw config env: `OPENCLAW_CONFIG_PATH`
- Agent workspace root (resolved at generation time): `C:/Users/Admin/AppData/Roaming/V-Claw/.openclaw/workspace/main`
- Workspace skills location: `<agent-workspace>/../skills`

## Runtime Rules

1. Treat the workspace root above as the primary location for agent project files.
2. Treat `.openclaw` state/config directories as runtime state storage, not project workspace.
3. Install or reference workspace skills under `workspace/skills/<skill-id>/SKILL.md`.
4. Resolve current values from env vars instead of assuming fixed absolute paths.

## Notes

- V-Claw may run in installed or portable mode.
- OpenClaw commands are launched with environment variables pointing to these managed paths.
